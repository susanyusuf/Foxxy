## 1. Basic Info
- Name: Fox Compass
- Short description (one or two sentences): A contributor-first platform that simplifies onboarding, issue triage, and release planning for open-source projects.
- Long description (detailed):
  Fox Compass provides a focused maintainer toolkit that reduces overhead by standardizing contributor onboarding, offering issue-ready tasks for newcomers, and providing release templates that teams can reuse. It targets early-stage open-source projects and maintainers who need pragmatic, low-effort systems to grow their contributor base while keeping quality and sustainability high.
- Tech stack: React, Vite, HTML/CSS (static frontend); lightweight backend-ready structure (Node.js-compatible APIs if needed).

## 2. Purpose
- Problem statement:
  Many projects stall because maintainers do operational work (onboarding, triage, releases) that is manual and inconsistent. Contributors often don't know where to start and abandon attempts.
- What we build:
  A minimal maintainership OS: onboarding checklists, labeled issue templates, release planners, and contributor guides that reduce friction for first-time contributors and make maintainer tasks repeatable.
- Key outcomes:
  - Faster first-contribution times
  - Clearer scope for community contributors
  - Predictable release cadence for projects

## 3. Stellar (Impact / Why this matters)
- Who benefits: New maintainers, small OSS teams, and contributors seeking clear entry points.
- Metrics of success (example):
  - Increase first-time contributor completion rate by 50% in three months
  - Reduce time-to-merge for community PRs by 30%
  - Maintain a backlog of 10 'good first issues' per release
- Long-term impact: Help projects move from maintainer burnout to a sustainable stewardship model with documented workflows.

## 4. Details (Implementation plan)
- Phase 1 (Prototype, 6–8 weeks):
  - Implement contributor checklist UI and issue board
  - Create issue templates and CONTRIBUTING guide
  - Add initial set of 4–8 curated contributor issues
- Phase 2 (Feedback, 8–10 weeks):
  - Run onboarding sessions with 5–10 early contributors

- Phase 3 (Launch, 4–6 weeks):
  - Polish docs, publish guides, and onboard first cohort
